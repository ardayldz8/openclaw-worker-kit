# README Growth Checklist

Goal: make the repo understandable in 30 seconds and credible in 3 minutes.

## 1) Above the fold (first screen)
- [ ] Clear one-line value proposition
- [ ] Problem statement (why not plain cron / why not heavy orchestrators)
- [ ] Primary CTA: Quickstart
- [ ] Secondary CTA: Architecture / Runbook

## 2) Quickstart (copy-paste)
- [ ] Install command block
- [ ] First job run command block
- [ ] Expected output snippet
- [ ] Uninstall/rollback command

## 3) Core features (short bullets)
- [ ] Retry/backoff
- [ ] Alert hooks
- [ ] Manifest + chain orchestration
- [ ] Checkpoint/resume
- [ ] Metrics/status/trend/history

## 4) Trust signals
- [ ] CI badge
- [ ] Latest release badge
- [ ] License badge
- [ ] Links to runbook/docs

## 5) SEO & discoverability
- [ ] Repo description updated
- [ ] Topics set: openclaw, worker, orchestration, systemd, devops, automation, self-hosted, bash
- [ ] Consistent keywords in README headings

## 6) Adoption helpers
- [ ] "Who is this for" section
- [ ] "When not to use" section
- [ ] Migration notes link (v0.4 -> v0.5)
- [ ] Contributing section and good-first-issues pointer

## 7) Media
- [ ] 30–60 sec terminal GIF (install + run + status)
- [ ] Optional architecture diagram

## Definition of done
- New user can run first successful demo in <10 minutes.
